# BouyomiSchedule Development Rules

## Coding Standards
- すべての制御ブロック（if, for, while 等）には必ず波括弧 `{}` を使用すること。
- マジックナンバーを避け、意味のある定数または `enum` を使用すること。
- Qt の命名規則（キャメルケース）に従い、メンバ変数は `m_` プリフィックスを付けること。
- 文字列リテラルには `QStringLiteral` または `tr()` を使用すること。

## Architecture
- UIロジックは `ui/`、主要なスケジューリングロジックは `core/`、外部連携は `api/` に分離すること。
- 棒読みちゃんとの通信は `BouyomiClient` クラスに集約すること。

## Documentation
- 公開メソッドには Doxygen 形式のコメントを付与すること。
- 設計変更時は `doc/detailed_design.md` を更新すること。
