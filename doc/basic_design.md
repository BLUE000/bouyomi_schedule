# 基本設計書: BouyomiSchedule (C++ Port)

## 1. システム構成
本システムは、C++ (Qt6) フレームワークを使用したデスクトップアプリケーションである。

### 1.1 モジュール構成
- **UI層 (`ui/`)**: ユーザーインターフェース。ユーザー入力を受け取り、タイマー状態を表示する。
- **コア層 (`core/`)**: タイマーの管理、スケジューリングロジック、設定ファイルの処理。
- **API層 (`api/`)**: 外部ソフトウェア（棒読みちゃん）との通信プロトコル実装。

## 2. 技術スタック
- **言語**: C++17
- **GUIフレームワーク**: Qt 6.10 (Widgets)
- **ビルドシステム**: CMake, Ninja
- **通信**: Qt Network (QTcpSocket, QNetworkAccessManager)
- **データ形式**: JSON (QJsonDocument)

## 3. ディレクトリ構造
```
BouyomiSchedule/
├── api/            # 棒読みちゃん通信クライアント
├── core/           # タイマー管理・ロジック
├── ui/             # GUI関連 (MainWindow等)
├── doc/            # 設計・要件ドキュメント
├── tests/          # ユニットテスト
├── bin/            # ビルド成果物 (EXE, DLL)
└── config/         # 設定ファイル (JSON)
```

## 4. データフロー
1.  **起動**: `TimerManager` が `config/` 内の JSON を読み込み、初期タイマーを設定。
2.  **監視**: `TimerManager` が 1秒周期で現在時刻とスケジュールを照合。
3.  **発火**: 時刻到達時、`BouyomiClient` を介してメッセージを送信。
4.  **UI更新**: 1秒周期で UI が `TimerManager` から状態を取得し、カウントダウンを更新。

## 5. 通信仕様
- **HTTP送信**: 簡易的な GET リクエストによる読み上げ依頼。
- **TCP送信**: 独自バイナリパケット（リトルエンディアン）による高速・詳細な読み上げ依頼。
